## ✨ Why Choose GitHub Stats Cards?

Unlike generic alternatives, we offer **5 completely unique themes** designed by developers, for developers:

- 🌃 **Cyberpunk** - Neon futuristic aesthetic with glow effects
- 🎯 **Brutalist** - Raw minimalist design with bold typography
- 💻 **Terminal** - Retro hacker style with green-on-black
- 👑 **Luxury** - Elegant gold accents on dark backgrounds
- 🌊 **Vaporwave** - 80s retro vibes with pastel gradients

**Plus:**
- ⚡ Real-time stats with smart 1-hour caching
- 🎭 Mix and match themes for creative combinations
- 🔓 No authentication or signup required
- 💯 100% free and open source (MIT License)
- 🚀 Fast SVG generation with CDN delivery

> [!TIP]
> Mix and match different themes for your stats and languages cards to create a unique profile look!

**Replace `YOUR_USERNAME`** with your GitHub username and choose your favorite theme!

## 🛠️ Local Development

> [!WARNING]
> You need a GitHub Personal Access Token to run this project locally.

### Prerequisites

- Node.js 18 or higher
- npm or yarn
- GitHub Personal Access Token

### Setup

1. **Clone the repository:**
```bash
git clone https://github.com/Emanuel0428/github-stats-cards.git
cd github-stats-cards
```

2. **Install dependencies:**
```bash
npm install
```

3. **Create a `.env` file:**
```env
PAT_1=your_github_token_here
PORT=3000
```

> [!IMPORTANT]
> Create a GitHub Personal Access Token at [GitHub Settings → Developer settings → Personal access tokens](https://github.com/settings/tokens)
> 
> **Required scopes:** `public_repo`, `read:user`

4. **Start the development server:**
```bash
npm start
```

The server will run at `http://localhost:3000`

> [!TIP]
> Use the interactive demo page at `http://localhost:3000` to test all themes with different usernames!

## 🎨 Customization

### Creating Custom Themes

Each theme is implemented in separate files for easy customization:
- `src/cards/stats-[theme].js` - Stats card for specific theme
- `src/cards/topLanguages-[theme].js` - Languages card for specific theme

**To create a new theme:**

1. Copy an existing theme file (e.g., `stats-brutalist.js`)
2. Modify colors, fonts, and layout
3. Import and add the new theme to `src/server.js`
4. Add the theme option to the HTML selector

> [!WARNING]
> Avoid using external font imports (like Google Fonts) in SVG files. Browsers block external resources when SVGs are embedded as images. Use system fonts instead.

**Customizable elements:**
- **Fonts:** System fonts or web-safe fonts
- **Colors:** Background, accent, text colors
- **Layout:** Positioning, spacing, decorations
- **Effects:** Gradients, patterns, borders

### Modifying Existing Themes

Simply edit the corresponding theme file in `src/cards/` to adjust colors, fonts, or layout.

## ❓ FAQ

### How often do stats update?
Stats are cached for 1 hour for optimal performance. After 1 hour, fresh data is automatically fetched from GitHub's API.

### Do I need a GitHub token?
No! Users don't need any authentication. The service handles API authentication internally.

### Can I use this in my portfolio website?
Absolutely! The cards work anywhere that supports markdown or HTML images.

### Can I customize the colors?
Currently, customization is theme-based. Custom color support is planned for future updates.

### Is it really free?
Yes, 100% free and open source under MIT License. Use it for personal or commercial purposes without restrictions.

### What data is collected?
Only public GitHub statistics are fetched. No personal data is stored or collected.

### Why are my cards not loading?
Common issues:
- Invalid username (check spelling)
- GitHub API rate limit (wait a few minutes)
- Cache issues (add `&t=timestamp` to bypass cache)

### Can I contribute?
Yes! Contributions are welcome. Check the [Contributing](#-contributing) section.

## 🐛 Troubleshooting

### Cards not loading?

> [!WARNING]
> If cards show errors or don't load, check these common issues:

1. **Invalid username** - Verify the GitHub username exists and is spelled correctly
2. **GitHub token expired** - Regenerate your Personal Access Token (for self-hosted)
3. **Rate limit exceeded** - Wait a few minutes or use a different token
4. **Cache issues** - Add `&t=timestamp` to the URL to bypass cache

### SVG not rendering in browser?

> [!CAUTION]
> SVGs embedded as `<img>` cannot load external resources.

**System fonts that work:**
- `Arial`, `Helvetica`, `sans-serif`
- `Georgia`, `Times New Roman`, `serif`
- `Courier New`, `monospace`
- `Verdana`, `Tahoma`

### Need more help?

> [!TIP]
> Open an issue on GitHub with:
> - Your username
> - The theme you're using
> - A screenshot of the error
> - Browser console logs (F12 → Console)
